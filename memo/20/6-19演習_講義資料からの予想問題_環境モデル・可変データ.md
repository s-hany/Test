## 1. 演習でまず使う読み方

### 1.1 生成した回数を見る

```racket
(define W1 (make-withdraw 100))
(define W2 (make-withdraw 100))
```

この場合は `make-withdraw` を**二回**呼んでいる。

- `W1` 用の `balance`
- `W2` 用の `balance`

が別々に作られる。状態は独立する。

一方で、

```racket
(define W (make-withdraw 100))
(define W1 W)
(define W2 W)
```

は `make-withdraw` を一回しか呼んでいない。

- `W1`
- `W2`

は同じ手続き `W` の別名であり、同じ `balance` を共有する。

### 1.2 呼び出しの括弧を二段階に分ける

```racket
((acc 'secret 'withdraw) 40)
```

は一気に読まない。

1. `(acc 'secret 'withdraw)` を評価して、`withdraw` 手続きを得る。
2. その手続きに `40` を渡す。

つまり、`dispatch` の `'withdraw` 節では `withdraw` を**実行するのではなく返す**。

```racket
((eq? m 'withdraw) withdraw)
```

### 1.3 `lambda` は作られた環境を覚える

```racket
(let ((x 1))
  (let ((f (lambda (y) (+ y x))))
    ...))
```

この `f` は、作られたときの `x = 1` を覚える。

あとから別の `x` を作っても、`f` の中の `x` がそちらに切り替わるわけではない。

# 2. 予想問題

<details>
<summary><strong>予想問題1（確定の土台）: make-monitored を自力で書く</strong></summary>

## 問題の型

**局所状態変数 + `set!` + メッセージ分岐型**

## 問題

一引数手続き `f` を受け取り、呼び出し回数を覚える手続き `make-monitored` を定義せよ。

- 数値など通常の入力なら、回数を1増やしてから `f` を実行する。
- `'how-many-calls?` なら、現在の回数を返す。
- `'reset-count` なら、回数を0に戻して0を返す。

```racket
(define s (make-monitored sqrt))

(s 100)              ; 10
(s 'how-many-calls?) ; 1
(s 81)               ; 9
(s 'how-many-calls?) ; 2
(s 'reset-count)     ; 0
(s 64)               ; 8
(s 'how-many-calls?) ; 1
```

## 解答

```racket
(define (make-monitored f)
  (let ((count 0))
    (lambda (x)
      (cond ((eq? x 'how-many-calls?)
             count)
            ((eq? x 'reset-count)
             (begin
               (set! count 0)
               count))
            (else
             (begin
               (set! count (+ count 1))
               (f x)))))))
```

## 重要な環境モデルの見方

```racket
(define s (make-monitored sqrt))
```

を評価した瞬間に、`count = 0` を含む環境が一つ作られる。返された `lambda` である `s` は、その環境を覚えている。

そのため、毎回 `count` を作り直すのではなく、

```text
0 → 1 → 2 → 0 → 1
```

と変化を覚えられる。

## よくあるミス

```racket
(define (make-monitored f)
  (lambda (x)
    (let ((count 0))
      ...)))
```

この形では、`s` を呼ぶたびに `count` が0から始まる。状態を覚えられない。

</details>

<summary><strong>予想問題2（高）: クロージャは 11 か 12 か</strong></summary>

## 問題の型

**クロージャ / 静的スコープ / 環境モデル型**

## 問題

次の式の値を答え、理由を説明せよ。

```racket
(let ((x 1))
  (let ((f (lambda (y) (+ y x))))
    (let ((x 2))
      (f 10))))
```

## 解答

```text
11
```

## 環境を順に追う

### A1: 最初の `let`

```racket
(let ((x 1))
  ...)
```

で、外側の環境に

```text
x = 1
```

が束縛される。

### A2: `f` を作る

```racket
(lambda (y) (+ y x))
```

が作られる。この手続きは、**作られた時点の環境 A1** を覚える。

つまり `f` の中にある `x` は、A1 の `x = 1` を指す。

### A3: 内側の `let`

```racket
(let ((x 2))
  (f 10))
```

で、新しい `x = 2` が作られる。

しかし、`f` は A1 を覚えているため、`f` の本体 `(+ y x)` にある `x` は A1 の `1` である。

```racket
(f 10)
→ (+ 10 1)
→ 11
```

## よくあるミス

内側に `x = 2` があるからといって、`f` の `x` が `2` になると考えないこと。

`f` は「呼ばれた場所」の環境ではなく、**作られた場所**の環境を覚える。

</details>

## 問題の型

**局所状態変数 + クロージャ型**

## 問題

初期残高 `balance` を受け取り、引き出し手続きを返す `make-withdraw` を定義せよ。

```racket
(define W (make-withdraw 100))

(W 50) ; 20
(W 30) ; "Insufficient funds"
```

## 解答

```racket
(define (make-withdraw balance)
  (lambda (amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))

        "Insufficient funds")))
```

最初は `balance = 100`。

```racket
(W 30)
```

で

```text
balance: 100 → 70
```

次に

```racket
(W 50)
```

で

```text
balance: 70 → 20
```

最後の `(W 30)` では `20 >= 30` が偽なので、残高を変更せず文字列を返す。

## よくあるミス

```racket
(set! balance amount)
```

ではなく、残高から引く。

```racket
(set! balance (- balance amount))
```

<summary><strong>予想問題7（高）: make-account を完成させる</strong></summary>

## 問題の型

**局所状態 + 内部定義 + dispatch 型**

## 問題

パスワード付き口座を作る `make-account` を定義せよ。

```racket
(define acc (make-account 100 'secret))

((acc 'secret 'withdraw) 40) ; 60
((acc 'secret 'deposit) 30)  ; 90
((acc 'wrong 'withdraw) 10)  ; "Incorrect password"
```

## 解答

```racket
(define (make-account balance password)
  (define (withdraw amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))
          balance)
        "Insufficient funds"))

  (define (deposit amount)
    (set! balance (+ balance amount))
    balance)

  (define (dispatch given-password m)
    (cond ((not (eq? given-password password))
           (lambda (amount)
             "Incorrect password"))
          ((eq? m 'withdraw)
           withdraw)
          ((eq? m 'deposit)
           deposit)
          (else
           (error "Unknown request -- MAKE-ACCOUNT" m))))

  dispatch)
```

## 二段階の評価

```racket
((acc 'secret 'withdraw) 40)
```

は次のように読む。

```racket
(acc 'secret 'withdraw)
```

が `withdraw` を返す。

```racket
(withdraw 40)
```

が残高を `100` から `60` に更新する。

## ひっかけ

間違ったパスワードのとき、`"Incorrect password"` を直接返してはいけない。

```racket
((acc 'wrong 'withdraw) 10)
```

では、最初の段階の `(acc 'wrong 'withdraw)` が、あとで `10` を受け取れる**手続き**を返す必要がある。

そのため、

```racket
(lambda (amount)
  "Incorrect password")
```

を返す。

</details>

<summary><strong>予想問題8（中〜高）: make-monitored の状態を一つ増やす</strong></summary>

## 問題の型

**複数の局所状態変数 + メッセージ分岐型**

## 問題

一引数手続き `f` を監視する `make-monitored-with-last` を定義せよ。

- 通常の入力では `f` を実行する
- `'how-many-calls?` で呼び出し回数を返す
- `'last-argument` で最後に `f` へ渡した引数を返す
- `'reset-count` で回数だけを0に戻す

```racket
(define (square x)
  (* x x))

(define s (make-monitored-with-last square))

(s 5)               ; 25
(s 'last-argument)  ; 5
(s 10)              ; 100
(s 'how-many-calls?) ; 2
```

## 解答

```racket
(define (make-monitored-with-last f)
  (let ((count 0)
        (last-argument 'none))
    (lambda (x)
      (cond ((eq? x 'how-many-calls?)
             count)
            ((eq? x 'last-argument)
             last-argument)
            ((eq? x 'reset-count)
             (begin
               (set! count 0)
               count))
            (else
             (begin
               (set! count (+ count 1))
               (set! last-argument x)
               (f x)))))))
```

## 3.2との対応

練習問題3.2と違うのは、状態が一つ増えただけである。

```racket
count
last-argument
```

どちらも `let` の中に置き、返した `lambda` が覚える。

<summary><strong>予想問題9（中）: rand の generate / reset</strong></summary>

## 問題の型

**局所状態 + メッセージパッシング + 手続きを返す型**

## 問題

疑似乱数生成器を考える。内部状態を `x` とし、

- `(rand 'generate)` は `x` を次の値に更新して返す
- `((rand 'reset) new-value)` は `x` を `new-value` にする

ように `rand` を定義せよ。

`random-init` と `rand-update` はすでに定義されているものとする。

## 解答

```racket
(define rand
  (let ((x random-init))
    (lambda (m)
      (cond ((eq? m 'generate)
             (begin
               (set! x (rand-update x))
               x))
            ((eq? m 'reset)
             (lambda (new-value)
               (set! x new-value)
               x))
            (else
             (error "Unknown request -- RAND" m))))))
```

## 最重要ポイント

```racket
((rand 'reset) 10)
```

という呼び方に注目する。

最初に評価されるのは、

```racket
(rand 'reset)
```

である。これは `10` を受け取れる手続きを返す必要がある。

だから `'reset` の分岐では、

```racket
(lambda (new-value)
  ...)
```

を返す。

</details>

<summary><strong>予想問題10（中）: 状態がある f と評価順序</strong></summary>

## 問題の型

**局所状態 + 評価順序 + 副作用型**

## 問題

最初の呼び出しだけ引数を返し、2回目以降は `0` を返す `f` を定義せよ。

```racket
(define f
  ...)

(let ((right (f 1)))
  (let ((left (f 0)))
    (+ left right)))
; 1
```

## 解答

```racket
(define f
  (let ((first? #t))
    (lambda (x)
      (if first?
          (begin
            (set! first? #f)
            x)
          0))))
```

## 評価の流れ

最初の `(f 1)` では `first?` が `#t`。

```racket
(set! first? #f)
```

を行ってから `1` を返す。

次の `(f 0)` は2回目なので `0` を返す。

```racket
(+ 0 1)
→ 1
```

## 注意

代入があると、部品を評価する順番が意味を持つ。

以前の「式の部分を置換して考えればよい」という理解だけでは不十分になる。これが、講義で環境モデルと `set!` を一緒に扱った理由である。

<summary><strong>予想問題11（中）: sum-of-squares を環境モデルで追う</strong></summary>

## 問題の型

**環境モデル / 仮引数束縛 / 手続き適用型**

## 問題

次の `(f 5)` を評価せよ。また、`a`、`x`、`y` がそれぞれ何になるか答えよ。

```racket
(define (square x)
  (* x x))

(define (sum-of-squares x y)
  (+ (square x)
     (square y)))

(define (f a)
  (sum-of-squares (+ a 1)
                  (* a 2)))

(f 5)
```

## 解答

```text
136
```

## 評価の順序

```racket
(f 5)
→ (sum-of-squares (+ 5 1) (* 5 2))
→ (sum-of-squares 6 10)
→ (+ (square 6) (square 10))
→ (+ 36 100)
→ 136
```

## 環境ごとの束縛

```text
f を適用した環境: a = 5
sum-of-squares を適用した環境: x = 6, y = 10
square を適用した環境: x = 6
square をもう一度適用した環境: x = 10
```

重要なのは、`f` の `a` と `sum-of-squares` の `x` は同じ変数ではないこと。

同じように見える名前でも、どの手続きの仮引数か、どの環境で束縛されたかを分けて考える。

</details>

<details>
<summary><strong>予想問題12（中）: 可変ペアを共有したときの set-car! の効果</strong></summary>

## 問題の型

**可変データ + 共有構造 + 副作用型**

## 問題（Scheme / SICP風の表記）

```racket
(define x (cons 1 2))
(define y x)

(set-car! x 10)

(car y)
```

の値を答えよ。

## 解答

```text
10
```

## 理由

```racket
(define y x)
```

は、ペアを複製するのではない。`x` と `y` が同じペアを指す。

そのため、

```racket
(set-car! x 10)
```

でペアの先頭部分を変更すると、`y` から見ても先頭は `10` になる。

```text
変更前: x と y → (1 . 2)
変更後: x と y → (10 . 2)
```

## Racketでの注意

通常の `#lang racket` では、`cons` が作るペアは変更できない。その場合は可変ペア用に以下を使う。

```racket
(define x (mcons 1 2))
(define y x)

(set-mcar! x 10)

(mcar y)
```

授業のスタータファイルに `set-car!` が現れた場合は、問題文・補助ライブラリの表記に合わせる。

</details>

<summary><strong>予想問題13（中）: set-cdr! で後ろを差し替える</strong></summary>

## 問題の型

**可変データ + リスト構造の変更型**

## 問題（Scheme / SICP風の表記）

```racket
(define x (cons 1 (cons 2 '())))

(set-cdr! x (cons 10 '()))

x
```

の結果を答えよ。

## 解答

```text
(1 10)
```

## 図で考える

最初は、

```text
x → [1 | ] → [2 | '()]
```

である。

```racket
(set-cdr! x (cons 10 '()))
```

は、最初のペアの `cdr` を `(10)` に差し替える。

```text
x → [1 | ] → [10 | '()]
```

よって結果は `(1 10)`。

## Racketでの対応

通常のRacketで可変ペアを使う場合は、

```racket
(define x (mcons 1 (mcons 2 '())))
(set-mcdr! x (mcons 10 '()))
x
```

という形になる。

<summary><strong>予想問題14（中）: set-car! か set-cdr! かを選ぶ穴埋め</strong></summary>

## 問題の型

**可変ペアの操作選択型**

## 問題

次のような可変ペア `p` の先頭の値を `new-first` に変更したい。`???` を埋めよ。

```racket
(define (replace-first! p new-first)
  (??? p new-first)
  p)
```

## 解答（Scheme / SICP風）

```racket
(define (replace-first! p new-first)
  (set-car! p new-first)
  p)
```

## 問題の型

**可変データ + 別名 + 副作用の波及型**

## 問題（Scheme / SICP風の表記）

```racket
(define p (cons 1 2))
(define q p)

(set-cdr! q 100)

(cons (car p) (cdr p))
```

の値を答えよ。

## 解答

```text
(1 . 100)
```

## 理由

`q` は `p` と同じペアを指す。

```racket
(set-cdr! q 100)
```

で同じペアの後半が `100` に変更される。

よって `p` を通しても、

```racket
(car p) ; 1
(cdr p) ; 100
```

となる。

## よくあるミス

`q` を別のペアだと考えないこと。

```racket
(define q p)
```

は複製ではない。

# 3. 自力で書くためのテンプレート

## 3.1 状態を覚える手続き

```racket
(define (make-something initial)
  (let ((state initial))
    (lambda (x)
      (set! state ...)
      state)))
```

使う場面:

- `make-accumulator`
- `make-withdraw`
- 呼び出し回数カウンタ
- 最後の入力を覚える手続き

---

## 3.2 特別な記号で動作を変える手続き

```racket
(define (make-something f)
  (let ((state initial))
    (lambda (x)
      (cond ((eq? x 'message-1)
             ...)
            ((eq? x 'message-2)
             ...)
            (else
             ...)))))
```

使う場面:

- `make-monitored`
- `rand`
- リセット付きカウンタ

---

## 3.3 手続きを返す dispatch

```racket
(define (make-object state)
  (define (operation-a x)
    ...)

  (define (operation-b x)
    ...)

  (define (dispatch m)
    (cond ((eq? m 'operation-a)
           operation-a)
          ((eq? m 'operation-b)
           operation-b)
          (else
           (error "Unknown request" m))))

  dispatch)
```

使う場面:

- `make-account`
- `make-joint`
- `'reset` が新しい値をさらに受け取る `rand`
