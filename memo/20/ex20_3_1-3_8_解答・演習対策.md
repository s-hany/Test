# 最初に：今回の範囲で一番大事な考え方

今回の問題は、ほとんどが次の一文にまとまる。

> **手続きが、前回までの値を局所変数として覚え、`set!` でその値を更新する。**

例えば、普通の手続きは過去を覚えない。

```racket
(define (add10 x)
  (+ x 10))

(add10 5) ; 15
(add10 5) ; 15
```

一方、今回の `make-accumulator` は前回までの合計を覚える。

```racket
(define A (make-accumulator 5))

(A 10) ; 15
(A 10) ; 25
```

このとき `A` の内部には、最初は `5`、1回目の後は `15`、2回目の後は `25` という状態が保存されている。

## 基本の骨組み

```racket
(define (make-something initial-state)
  (let ((state initial-state))
    (lambda (input)
      (set! state ...)
      ...)))
```

- `let` の中の `state` が、外から直接触れない**局所状態変数**。
- `lambda` が返されても、`state` は消えずに残る。
- `set!` は、すでにある `state` の中身を変更する。
- `define` は新しい名前を定義するもの。**値の更新には使わない。**

## テストコードの形から返り値を読む

今回の演習で最も重要な読み方。

| テストの呼び方 | 何を意味するか | 作るべき形 |
|---|---|---|
| `(A 10)` | `A` は数を1個受け取る手続き | `(lambda (x) ...)` |
| `(s 'how-many-calls?)` | `s` は記号メッセージを受け取る手続き | `(lambda (m) (cond ...))` |
| `((acc 'withdraw) 40)` | `(acc 'withdraw)` が手続きを返す | `dispatch` が `withdraw` を返す |
| `((acc 'pass 'withdraw) 40)` | まずパスワードと操作名を受け取り、次に金額を受け取る | `(define (dispatch pass m) ...)` |
| `((rand 'reset) 3)` | `(rand 'reset)` が1引数手続きを返す | `'reset` で `(lambda (new-value) ...)` を返す |

> `((acc 'withdraw) 40)` を見たら、`withdraw` はまだ実行せず、**手続きそのものを返す**必要がある。

# 3.1 / ex20-1：make-accumulator

<details>
<summary><strong>問題の内容・型・解答・テスト対応を開く</strong></summary>

## 問題の型

**局所状態変数 + `set!` 型**。

教科書の練習問題3.1は、初期値を受け取って、以後の入力を合計し続けるアキュムレータを作る問題である。

```racket
(define A (make-accumulator 5))
(A 10) ; 15
(A 10) ; 25
```

## 何を覚えるか

```text
sum = 今までの合計
```

最初は `sum = 5`。  
`(A 10)` を呼ぶと `sum = 15`。  
もう一度 `(A 10)` を呼ぶと `sum = 25`。

## `???` に貼る解答

```racket
(let ((sum num))
  (lambda (x)
    (set! sum (+ sum x))
    sum))
```

## 手続き全体の完成形

```racket
(define (make-accumulator num)
  (let ((sum num))
    (lambda (x)
      (set! sum (+ sum x))
      sum)))
```

## テストが通る流れ

```racket
(define A (make-accumulator 5))
```

この時点で、`A` は `sum = 5` を覚えている手続きになる。

```racket
(A 10)
```

内部では次が起こる。

```racket
(set! sum (+ sum 10))
```

したがって `sum` は `5` から `15` に変わり、`15` を返す。

次の `(A 10)` は、`sum = 15` から始まるため、`25` を返す。

## よくあるミス

### ミス1：`define` で更新する

```racket
(define sum (+ sum x))
```

これは値の更新ではない。今回必要なのは、すでにある `sum` の値を変えることなので、正しくは次。

```racket
(set! sum (+ sum x))
```

### ミス2：`sum` を `lambda` の中で作る

```racket
(lambda (x)
  (let ((sum num))
    ...))
```

これだと、呼び出すたびに `sum` が初期値に戻る。  
`sum` は、返す `lambda` の**外側**に置く。

## 自力で書くための空欄骨組み

```racket
(define (make-accumulator num)
  (let ((sum ___))
    (lambda (x)
      (set! sum (___ sum x))
      ___)))
```

</details>

# 3.2 / ex20-2：make-monitored

<details>
<summary><strong>今回の本命：問題の型・完全解答・評価の流れを開く</strong></summary>

## 問題の型

**局所状態変数 + 記号メッセージによる分岐 + `set!` 型**。

教科書の練習問題3.2は、1引数の手続き `f` を監視し、何回普通の入力で呼ばれたかを数える手続きを作る問題である。

```racket
(define s (make-monitored sqrt))

(s 100)              ; 10
(s 'how-many-calls?) ; 1
(s 81)               ; 9
(s 'how-many-calls?) ; 2
(s 'reset-count)     ; 0 に戻す
```

## 何を覚えるか

```text
count = f を実際に呼んだ回数
```

- `'how-many-calls?` は回数を聞くだけなので増やさない。
- `'reset-count` は回数を0に戻すだけなので増やさない。
- それ以外の入力でだけ `count` を1増やし、`(f x)` を実行する。

## `???` に貼る解答

```racket
(let ((count 0))
  (lambda (x)
    (cond ((eq? x 'how-many-calls?)
           count)
          ((eq? x 'reset-count)
           (begin
             (set! count 0)
             count))
          (else
           (begin
             (set! count (+ count 1))
             (f x))))))
```

## 手続き全体の完成形

```racket
(define (make-monitored f)
  (let ((count 0))
    (lambda (x)
      (cond ((eq? x 'how-many-calls?)
             count)
            ((eq? x 'reset-count)
             (begin
               (set! count 0)
               count))
            (else
             (begin
               (set! count (+ count 1))
               (f x)))))))
```

## なぜ `eq?` か

`'how-many-calls?` と `'reset-count` は数値ではなく**記号**である。

```racket
(eq? x 'how-many-calls?)
```

は「`x` はこの記号と同じか」を調べている。

```racket
(= x 'how-many-calls?)
```

は使えない。`=` は数値比較のためである。

## テストの評価を追う

```racket
(define s (make-monitored sqrt))
```

ここで、`s` は次を覚えた手続きになる。

```text
f = sqrt
count = 0
```

### 1回目

```racket
(s 100)
```

`100` は特別な記号ではないため `else` に入る。

```racket
(set! count (+ count 1)) ; count は 1
(f x)                    ; (sqrt 100) → 10
```

結果は `10`。

### 回数の確認

```racket
(s 'how-many-calls?)
```

`count` を返すだけなので、結果は `1`。  
この操作自体は `sqrt` を呼ばないため、回数は増えない。

### リセット後

```racket
(s 'reset-count)
(s 64)
(s 'how-many-calls?)
```

リセットで `count = 0`。  
`(s 64)` で `count = 1`。  
最後の問い合わせは `1` を返す。

## よくあるミス

### ミス1：カウンタの場所が内側すぎる

```racket
(define (make-monitored f)
  (lambda (x)
    (let ((count 0))
      ...)))
```

これは毎回 `count = 0` を作るため、回数を覚えられない。

正しくは、`let` を `lambda` の外側に置く。

```racket
(define (make-monitored f)
  (let ((count 0))
    (lambda (x)
      ...)))
```

### ミス2：特別な記号にクォートをつけない

```racket
(s how-many-calls?)
```

これは `how-many-calls?` という変数を探しに行く。

```racket
(s 'how-many-calls?)
```

と、記号として渡す。

### ミス3：普通の入力より先に `(f x)` を実行する

`'how-many-calls?` や `'reset-count` に対して `sqrt` などを呼んではいけない。  
必ず `cond` で特別な入力を先に分ける。

## 自力で書くための空欄骨組み

```racket
(define (make-monitored f)
  (let ((count 0))
    (lambda (x)
      (cond ((eq? x 'how-many-calls?)
             ___)
            ((eq? x 'reset-count)
             (begin
               (set! count ___)
               ___))
            (else
             (begin
               (set! count (___ count 1))
               (___ x)))))))
```

</details>

# 3.3 / ex20-3：パスワード付き make-account

<details>
<summary><strong>dispatch が「手続きそのもの」を返す理由まで開く</strong></summary>

## 問題の型

**局所状態変数 + 内部手続き + dispatch 型**。

教科書の元の口座では、`balance` を内部状態として持ち、`withdraw` と `deposit` を返す。  
練習問題3.3では、その口座にパスワードを追加する。

```racket
(define acc (make-account 100 'secret-password))

((acc 'secret-password 'withdraw) 40)
; 60

((acc 'some-other-password 'deposit) 50)
; "Incorrect password"
```

## テストの形を分解する

```racket
((acc 'secret-password 'withdraw) 40)
```

これは一度に読まない。

### 第1段階

```racket
(acc 'secret-password 'withdraw)
```

この結果は、次に `40` を受け取る必要がある。  
したがって、ここで返すものは数値ではなく**手続き**でなければならない。

### 第2段階

```racket
(withdraw 40)
```

ここで残高を更新する。

## `???` に貼る解答

```racket
(let ()
  (define (withdraw amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))
          balance)
        "Insufficient funds"))

  (define (deposit amount)
    (set! balance (+ balance amount))
    balance)

  (define (dispatch given-pass m)
    (cond ((not (eq? given-pass pass))
           (lambda (amount) "Incorrect password"))
          ((eq? m 'withdraw)
           withdraw)
          ((eq? m 'deposit)
           deposit)
          (else
           (error "Unknown request -- MAKE-ACCOUNT" m))))

  dispatch)
```

## 提出向けに見やすい完成形

`let` はなくてもよい。Racketでは、手続き本体に内部 `define` を書ける。

```racket
(define (make-account balance pass)
  (define (withdraw amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))
          balance)
        "Insufficient funds"))

  (define (deposit amount)
    (set! balance (+ balance amount))
    balance)

  (define (dispatch given-pass m)
    (cond ((not (eq? given-pass pass))
           (lambda (amount) "Incorrect password"))
          ((eq? m 'withdraw)
           withdraw)
          ((eq? m 'deposit)
           deposit)
          (else
           (error "Unknown request -- MAKE-ACCOUNT" m))))

  dispatch)
```

## なぜ間違ったパスワードでも `lambda` を返すのか

間違ったパスワードでのテストは次。

```racket
((acc 'some-other-password 'deposit) 50)
```

もし第1段階で文字列を直接返すと、全体は次のようになってしまう。

```racket
("Incorrect password" 50)
```

文字列を手続きとして呼ぼうとするためエラーになる。

そのため、次のような「金額を受け取ってエラーメッセージを返す手続き」を返す。

```racket
(lambda (amount) "Incorrect password")
```

すると、

```racket
((lambda (amount) "Incorrect password") 50)
```

となり、正しく文字列が返る。

## よくあるミス

### ミス1：`withdraw` をここで呼んでしまう

```racket
((eq? m 'withdraw) (withdraw))
```

これはダメ。まだ `amount` が届いていない。

正しくは、手続きそのものを返す。

```racket
((eq? m 'withdraw) withdraw)
```

### ミス2：`balance` を引数に追加してしまう

`balance` は外から毎回渡す値ではなく、口座が内部に保持する状態。  
`withdraw` や `deposit` の引数は `amount` だけでよい。

### ミス3：`eq?` の左右を混同する

```racket
(eq? given-pass pass)
```

で、渡されたパスワードと作成時のパスワードを比べる。

</details>

# 3.4 / ex20-4：7回連続失敗で call-the-cops

<details>
<summary><strong>「合計7回」ではなく「7回連続」が重要な問題を開く</strong></summary>

## 問題の型

**make-account + 状態変数が2つある型**。

保存すべき状態は次の2つ。

```text
balance      = 口座残高
wrong-count  = 連続してパスワードを間違えた回数
```

## 問題の重要点

教科書では「7回連続でパスワードが間違っていた場合」とある。  
したがって、正しいパスワードで操作できたら `wrong-count` は0に戻す必要がある。

zipのテストも、4回間違えたあとに正しい `deposit` を行い、その後に改めて7回間違える流れになっている。

## `???` に貼る解答

```racket
(let ()
  (define wrong-count 0)

  (define (withdraw amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))
          balance)
        "Insufficient funds"))

  (define (deposit amount)
    (set! balance (+ balance amount))
    balance)

  (define (wrong-password amount)
    (set! wrong-count (+ wrong-count 1))
    (if (>= wrong-count 7)
        (call-the-cops)
        "Incorrect password"))

  (define (dispatch given-pass m)
    (cond ((not (eq? given-pass pass))
           wrong-password)
          ((eq? m 'withdraw)
           (begin
             (set! wrong-count 0)
             withdraw))
          ((eq? m 'deposit)
           (begin
             (set! wrong-count 0)
             deposit))
          (else
           (error "Unknown request -- MAKE-ACCOUNT" m))))

  dispatch)
```

## 完成形

```racket
(define (make-account balance pass)
  (define wrong-count 0)

  (define (withdraw amount)
    (if (>= balance amount)
        (begin
          (set! balance (- balance amount))
          balance)
        "Insufficient funds"))

  (define (deposit amount)
    (set! balance (+ balance amount))
    balance)

  (define (wrong-password amount)
    (set! wrong-count (+ wrong-count 1))
    (if (>= wrong-count 7)
        (call-the-cops)
        "Incorrect password"))

  (define (dispatch given-pass m)
    (cond ((not (eq? given-pass pass))
           wrong-password)
          ((eq? m 'withdraw)
           (begin
             (set! wrong-count 0)
             withdraw))
          ((eq? m 'deposit)
           (begin
             (set! wrong-count 0)
             deposit))
          (else
           (error "Unknown request -- MAKE-ACCOUNT" m))))

  dispatch)
```

## テスト対応

最初の4回の間違いでは、毎回次が起きる。

```racket
(set! wrong-count (+ wrong-count 1))
```

したがって、`wrong-count` は `1, 2, 3, 4` と進み、毎回 `"Incorrect password"` を返す。

その後の正しいパスワードによる操作で、次が実行される。

```racket
(set! wrong-count 0)
```

そのため、次の7回の誤入力は新しく数え直される。

7回目のとき、

```racket
(>= wrong-count 7)
```

が真になり、`(call-the-cops)` の結果、`"Call the cops!"` を返す。

## なぜ `wrong-password` は `amount` を受け取るのか

間違ったパスワードのときも、外側の呼び方は次のまま。

```racket
((acc 'wrong-password 'withdraw) 90)
```

`(acc 'wrong-password 'withdraw)` は、次に `90` を受け取る手続きを返す必要がある。  
そのため、`wrong-password` は実際には金額を使わなくても、形式上 `amount` を引数に持つ。

```racket
(define (wrong-password amount)
  ...)
```

## よくあるミス

### ミス1：正しいパスワードで失敗回数をリセットしない

これでは「7回連続」ではなく「累計7回」になってしまう。

### ミス2：`wrong-count` を `dispatch` の中で作る

```racket
(define (dispatch given-pass m)
  (define wrong-count 0)
  ...)
```

これではアクセスのたびに `wrong-count` が0に戻る。  
`wrong-count` は `make-account` の内部に1回だけ作る。

</details>

# 3.5 / ex20-5：Monte Carlo 積分と π の推定

<details>
<summary><strong>確率で面積を推定する高階手続き問題を開く</strong></summary>

## 問題の型

**高階手続き + ランダム試行 + 面積推定型**。

3.1の状態問題と少し見た目は違うが、`monte-carlo` に「1回の実験をする手続き」を渡す点で、高階手続きの復習にもなっている。

## 考え方

長方形の中からランダムに点を選ぶ。

- 点が対象領域の内側に入った回数を数える。
- 内側に入った割合を求める。
- その割合に長方形全体の面積をかける。

つまり、

```text
推定面積
= 長方形の面積 × (内側に入った回数 / 試行回数)
```

`monte-carlo` はすでに「成功した割合」を返すので、`estimate-integral` は長方形の面積を掛ければよい。

## `???` に貼る解答：estimate-integral

```racket
(* (- x2 x1)
   (- y2 y1)
   (monte-carlo trials
                (lambda ()
                  (P (random-in-range x1 x2)
                     (random-in-range y1 y2)))))
```

## `???` に貼る解答：estimate-pi

```racket
(let ()
  (define (square x)
    (* x x))

  (define (inside-unit-circle? x y)
    (<= (+ (square x)
           (square y))
        1.0))

  (estimate-integral inside-unit-circle?
                     -1.0 1.0
                     -1.0 1.0
                     trials))
```

## 完成形

```racket
(define (estimate-integral P x1 x2 y1 y2 trials)
  (* (- x2 x1)
     (- y2 y1)
     (monte-carlo trials
                  (lambda ()
                    (P (random-in-range x1 x2)
                       (random-in-range y1 y2))))))

(define (estimate-pi trials)
  (define (square x)
    (* x x))

  (define (inside-unit-circle? x y)
    (<= (+ (square x)
           (square y))
        1.0))

  (estimate-integral inside-unit-circle?
                     -1.0 1.0
                     -1.0 1.0
                     trials))
```

## なぜこれで π になるのか

半径1の円の面積は π。  
この円を含む長方形 `[-1, 1] × [-1, 1]` の面積は `4`。

```text
円に入る割合 ≒ π / 4
```

よって、

```text
4 × 円に入る割合 ≒ π
```

になる。

## テストコードとの対応

zipのテストは、値を表示するだけである。

```racket
(displayln (estimate-pi 10))
(displayln (estimate-pi 100))
(displayln (estimate-pi 1000))
...
```

乱数を使うので、結果は毎回同じではない。  
ただし、一般には試行回数が大きいほど π に近い値になりやすい。

## よくあるミス

### ミス1：述語 `P` を手続きとして呼ばない

```racket
(P x y)
```

の形で、ランダムに作った点が対象領域に入っているかを調べる。

### ミス2：面積を掛け忘れる

`monte-carlo` が返すのは面積ではなく「成功割合」。  
最後に長方形の面積を掛ける必要がある。

### ミス3：`random-in-range` を固定値で呼ぶ

毎回、独立したランダムな `x`, `y` を作る。

```racket
(random-in-range x1 x2)
(random-in-range y1 y2)
```

</details>

# 3.6 / ex20-6：reset できる rand

<details>
<summary><strong>`(rand 'generate)` と `((rand 'reset) 3)` の違いを開く</strong></summary>

## 問題の型

**局所状態変数 + 記号メッセージ + 手続きを返す分岐型**。

保存する状態は乱数列の現在値 `x`。

```racket
(rand 'generate)
```

なら、現在の `x` から次の値を作って返す。

```racket
((rand 'reset) 3)
```

なら、内部状態を `3` に戻す。

## 一番大事な読み方

```racket
((rand 'reset) 3)
```

は次の2段階。

```racket
(rand 'reset)
```

が「新しい初期値を受け取る手続き」を返す。

```racket
(lambda (new-value) ...)
```

その返された手続きに `3` を渡す。

## `???` に貼る解答

```racket
(let ((x random-init))
  (lambda (m)
    (cond ((eq? m 'generate)
           (begin
             (set! x (rand-update x))
             x))
          ((eq? m 'reset)
           (lambda (new-value)
             (set! x new-value)
             x))
          (else
           (error "Unknown request -- RAND" m)))))
```

## 完成形

```racket
(define rand
  (let ((x random-init))
    (lambda (m)
      (cond ((eq? m 'generate)
             (begin
               (set! x (rand-update x))
               x))
            ((eq? m 'reset)
             (lambda (new-value)
               (set! x new-value)
               x))
            (else
             (error "Unknown request -- RAND" m))))))
```

## テストが通る理由

```racket
((rand 'reset) 3)
```

で `x = 3` にする。

次に5回 `generate` して、`r1`〜`r5` を保存する。

もう一度、

```racket
((rand 'reset) 3)
```

で同じ状態 `x = 3` に戻す。

`rand-update` は、同じ入力なら同じ出力を返す。  
したがって、そこから再び生成される `r1`〜`r5` は最初の列と一致する。

## よくあるミス

### ミス1：`reset` で数値を直接返してしまう

```racket
((eq? m 'reset)
 (set! x new-value))
```

この場所には `new-value` が存在しない。  
しかも `(rand 'reset)` は次に値を受け取る手続きを返す必要がある。

### ミス2：`x` を `lambda` の内側で作る

`x` を呼び出すたびに作り直すと、乱数列が進まない。  
必ず外側の `let` に置く。

</details>

# 3.7 / ex20-7：make-joint

<details>
<summary><strong>新しい口座を作らず「同じ口座への別入口」を作る問題を開く</strong></summary>

## 問題の型

**既存の状態オブジェクトを共有する代理（wrapper）型**。

`make-joint` は、新しい残高を持つ別口座を作る問題ではない。  
すでにある口座 `acc` に対して、新しいパスワードでもアクセスできる入口を作る問題である。

```racket
(define peter-acc (make-account 100 'open-sesame))

(define paul-acc
  (make-joint peter-acc 'open-sesame 'rosebud))
```

ここで `peter-acc` と `paul-acc` は、同じ `balance` を共有する。

## テストから分かる条件

```racket
((peter-acc 'open-sesame 'withdraw) 40) ; 60
((paul-acc 'rosebud 'withdraw) 30)      ; 30
((paul-acc 'rosebud 'deposit) 50)       ; 80
((peter-acc 'open-sesame 'withdraw) 10) ; 70
```

最後の `70` が重要。  
`paul-acc` の操作が `peter-acc` の残高にも反映されている必要がある。

## `???` に貼る解答

```racket
(let ((challenge ((acc pass 'withdraw) 0)))
  (if (number? challenge)
      (lambda (given-pass m)
        (if (eq? given-pass new-pass)
            (acc pass m)
            (lambda (amount) "Incorrect password")))
      challenge))
```

## 完成形

```racket
(define (make-joint acc pass new-pass)
  (let ((challenge ((acc pass 'withdraw) 0)))
    (if (number? challenge)
        (lambda (given-pass m)
          (if (eq? given-pass new-pass)
              (acc pass m)
              (lambda (amount) "Incorrect password")))
        challenge)))
```

## なぜ `withdraw 0` を使って元パスワードを確認できるのか

```racket
((acc pass 'withdraw) 0)
```

- `pass` が正しければ、0円引き出しなので残高（数値）が返る。
- `pass` が間違っていれば、`"Incorrect password"` が返る。

そのため、

```racket
(number? challenge)
```

で正しいパスワードかを判定できる。

## なぜ `(acc pass m)` とするのか

`paul-acc` に新しいパスワードが来たら、元の口座に対して元のパスワード `pass` を使って、操作 `m` を転送する。

```racket
(acc pass m)
```

つまり、`paul-acc` は自分の `balance` を持たず、元の `acc` の `withdraw` / `deposit` をそのまま取り出して返す。

これにより、状態が共有される。

## テスト対応

### 元のパスワードを間違えた場合

```racket
(make-joint peter-acc 'open-file 'rosebud)
```

`'open-file` は誤りなので、`challenge` は `"Incorrect password"`。  
よって `make-joint` 自体がその文字列を返す。

### joint 口座に元のパスワードを渡した場合

```racket
((paul-acc 'open-sesame 'withdraw) 30)
```

`paul-acc` が受け付けるのは `new-pass` の `'rosebud` だけ。  
したがって `"Incorrect password"` を返す。

## よくあるミス

### ミス1：`make-account` をもう一度呼ぶ

```racket
(make-account 100 new-pass)
```

これは別の `balance` を持つ新しい口座を作ってしまう。  
共同口座ではない。

### ミス2：新しいパスワードで元の `acc` を直接呼ぶ

```racket
(acc new-pass m)
```

元の口座は `pass` しか知らないので、これは必ず失敗する。  
元口座を呼ぶときは、必ず元のパスワード `pass` を使う。

### ミス3：間違ったパスワードで文字列を直接返す

この問題でも呼び方は

```racket
((paul-acc 'wrong 'withdraw) 30)
```

なので、次に金額を受け取る手続きを返す必要がある。

</details>

# 3.8 / ex20-8：評価順序で値が変わる f

<details>
<summary><strong>「最初の1回だけ引数を返す」状態手続きを開く</strong></summary>

## 問題の型

**局所状態 + 評価順序型**。

教科書の狙いは、代入があると、

```racket
(+ (f 0) (f 1))
```

の2つの引数をどちらから先に評価するかで結果が変わり得る、ということを確かめること。

必要な `f` の振る舞いは、次のようなもの。

- 最初に呼ばれたときだけ、引数をそのまま返す。
- 2回目以降は常に `0` を返す。

すると、

```text
左から右： (f 0) が最初 → 0、(f 1) は2回目 → 0、合計 0
右から左： (f 1) が最初 → 1、(f 0) は2回目 → 0、合計 1
```

## `???` に貼る解答

```racket
(let ((first? #t))
  (lambda (x)
    (if first?
        (begin
          (set! first? #f)
          x)
        0)))
```

## 完成形

```racket
(define f
  (let ((first? #t))
    (lambda (x)
      (if first?
          (begin
            (set! first? #f)
            x)
          0))))
```

## zipのテストが少し特殊な理由

Racketにおける一般の手続き引数の評価順序には依存したくないため、zipのテストは `let` を使って順番を明示している。

```racket
(let ((r (f 1)))
  (let ((l (f 0)))
    (+ l r)))
```

この場合、必ず `(f 1)` が先。

- 最初の呼び出しなので `1` を返す。
- 次の `(f 0)` は2回目なので `0` を返す。
- `(+ l r)` は `(+ 0 1)` で `1`。

次のテストでは、`f` はすでに1回使われた状態のまま。

```racket
(let ((l (f 0)))
  (let ((r (f 1)))
    (+ l r)))
```

どちらも2回目以降の呼び出しなので `0`。  
結果は `0`。

## よくあるミス

### ミス1：`first?` を呼び出しごとに作る

```racket
(define f
  (lambda (x)
    (let ((first? #t))
      ...)))
```

これでは毎回 `first?` が `#t` に戻るため、常に最初の呼び出し扱いになる。

### ミス2：最初の呼び出しでも `0` を返してしまう

最初の呼び出しだけは `x` を返す必要がある。

```racket
(begin
  (set! first? #f)
  x)
```

の最後の `x` が重要。

</details>
